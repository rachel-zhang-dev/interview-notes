## 一、学习路径总览

```
基础夯实         核心场景专题            系统设计          面试冲刺
  ↓                  ↓                    ↓                ↓
4 周              8 周                  8 周             4 周
计网/OS/数据库     秒杀/订单/支付         架构设计         模拟面试
缓存/消息队列      库存/搜索/营销         高可用方案       八股 + 项目
```

### 核心知识地图

| 层次 | 关键技术 |
|------|---------|
| **存储层** | MySQL 索引/事务/锁、Redis 数据结构、ES 倒排索引、MongoDB |
| **缓存层** | 多级缓存、缓存穿透/击穿/雪崩、热 key、一致性 |
| **中间件** | Kafka/RocketMQ、Nginx、Zookeeper、Sentinel |
| **服务层** | 微服务（Spring Cloud / Dubbo）、RPC、服务治理 |
| **网关层** | API Gateway、限流、鉴权、灰度发布 |
| **架构** | 分布式事务、CAP/BASE、DDD、读写分离、分库分表 |
| **运维** | Docker、K8s、Prometheus、ELK、链路追踪 |

## 二、基础夯实阶段（1-4 周）

### 2.1 计算机基础（贯穿始终）

- [ ] **网络协议**
  - TCP 三次握手 / 四次挥手、流量控制、拥塞控制
  - HTTP 1.1 / 2 / 3 区别、HTTPS 原理、TLS 握手
  - WebSocket、长连接 vs 短连接
- [ ] **操作系统**
  - 进程 / 线程 / 协程、上下文切换、IPC
  - I/O 模型（BIO / NIO / AIO、epoll、Reactor）
  - 内存管理、虚拟内存、Page Cache
- [ ] **数据结构**
  - 跳表、布隆过滤器、一致性哈希、LRU/LFU
  - 红黑树 / B+ 树（数据库索引基础）

### 2.2 MySQL（必须吃透）

- [ ] **索引**
  - B+ 树原理、聚簇索引 vs 二级索引、回表、覆盖索引
  - 最左前缀匹配、索引失效场景
  - `EXPLAIN` 分析、慢 SQL 优化
- [ ] **事务**
  - ACID、四种隔离级别、幻读 vs 不可重复读
  - MVCC 原理、undo log / redo log / binlog
- [ ] **锁**
  - 表锁 / 行锁、共享锁 / 排他锁、间隙锁、Next-Key Lock
  - 死锁产生与排查
- [ ] **架构**
  - 主从复制原理、读写分离、分库分表
  - 主从延迟、半同步复制

**推荐资源**：《MySQL 是怎样运行的》、极客时间《MySQL 实战 45 讲》

### 2.3 Redis（电商核心）

- [ ] **5 种基础数据结构**：String / List / Hash / Set / ZSet
- [ ] **高级数据结构**：Bitmap / HyperLogLog / Geo / Stream
- [ ] **持久化**：RDB vs AOF、混合持久化
- [ ] **高可用**：主从、哨兵、Cluster 分片
- [ ] **典型问题**
  - 缓存穿透 / 击穿 / 雪崩
  - 大 key / 热 key 问题
  - 数据一致性方案（旁路缓存、延时双删、Canal）
  - 分布式锁（SETNX、Redisson、RedLock）

**推荐资源**：《Redis 设计与实现》、《Redis 深度历险》

### 2.4 消息队列

- [ ] **核心概念**：生产者 / 消费者 / Topic / Partition / 消费组
- [ ] **Kafka vs RocketMQ vs RabbitMQ** 对比
- [ ] **关键问题**
  - 消息可靠性（不丢、不重、有序）
  - 消息堆积处理
  - 事务消息、延迟消息
  - 顺序消息实现

**推荐资源**：《RocketMQ 技术内幕》、Kafka 官方文档

## 三、电商核心场景专题（5-12 周）

> 这一部分是电商面试的"主战场"，每个场景都要做到能完整画出架构图 + 写出关键代码。

### 3.1 商品系统（第 5 周）

**业务特点**：读多写少、SKU 复杂、需要搜索

- [ ] 商品分类树设计（多级分类、闭包表）
- [ ] SKU / SPU 数据模型
- [ ] 商品详情页性能优化
  - 多级缓存（本地缓存 Caffeine + Redis + CDN）
  - 静态化（页面静态化、对象静态化）
- [ ] 商品搜索
  - Elasticsearch 倒排索引、分词器
  - MySQL 与 ES 数据同步（Canal、双写）

**面试题示例**：
- 千万级商品的详情页如何做到 50ms 响应？
- 商品上下架后缓存如何更新？
- 商品搜索的相关性排序怎么实现？

### 3.2 购物车系统（第 6 周）

- [ ] 已登录 vs 未登录的购物车设计
- [ ] Redis Hash 存储购物车结构
- [ ] 购物车与商品价格 / 库存的实时校验
- [ ] 跨端同步（Web / App / 小程序）

### 3.3 库存系统（第 7 周）⭐核心

**业务特点**：高并发写、强一致性、防超卖

- [ ] 库存扣减方案对比
  - 下单减库存 vs 支付减库存 vs 预扣库存
- [ ] 防超卖方案
  - 数据库乐观锁（`WHERE stock > 0`）
  - 数据库悲观锁（`SELECT FOR UPDATE`）
  - Redis 原子操作（Lua 脚本）
  - 分布式锁
- [ ] 库存预热与回滚
- [ ] 多仓库存调度

**面试题示例**：
- 如何保证不超卖？三种方案的优劣？
- Redis 扣库存如何保证和数据库一致？
- 大促时库存扣减性能瓶颈怎么解决？

### 3.4 订单系统（第 8 周）⭐核心

- [ ] 订单状态机设计
- [ ] 订单号生成（雪花算法、号段模式）
- [ ] 订单分库分表
  - 分片键选择（用户 ID vs 订单 ID）
  - 跨库查询方案（ES 异构、基因法）
- [ ] 订单超时关闭
  - 定时任务扫表（不推荐）
  - 延迟队列（RocketMQ / Redis ZSet）
  - 时间轮算法
- [ ] 历史订单归档

**面试题示例**：
- 订单 ID 生成方案对比？
- 用户既要看自己的订单，商家又要看店铺订单，怎么分库分表？
- 千万级订单超时关闭怎么设计？

### 3.5 支付系统（第 9 周）⭐核心

**业务特点**：强一致性、资金安全、高可用

- [ ] 支付流程设计（统一收单、多渠道路由）
- [ ] 支付回调处理
  - 幂等性（防重表、状态机）
  - 异步通知与主动查询结合
- [ ] 对账系统设计
- [ ] 退款流程
- [ ] 分布式事务方案
  - 2PC / 3PC
  - TCC（Try-Confirm-Cancel）
  - Saga
  - 本地消息表
  - 最终一致性（MQ 事务消息）

**面试题示例**：
- 支付回调如何保证幂等？
- 用户付了钱但回调失败怎么办？
- 跨服务的订单 + 库存 + 支付，事务怎么保证？

### 3.6 秒杀系统（第 10 周）⭐⭐核心中的核心

**这是电商面试最高频的题目，必须烂熟于心**

- [ ] **流量分层削峰**
  - CDN 静态化
  - Nginx 限流
  - 网关层限流（Sentinel）
  - 应用层限流（令牌桶、漏桶、计数器、滑动窗口）
- [ ] **库存层防超卖**
  - Redis 预扣 + Lua 脚本
  - 异步落库（MQ）
- [ ] **防刷防作弊**
  - 验证码 / 滑动拼图
  - 用户限购
  - IP 黑名单
  - 接口签名
- [ ] **下单优化**
  - 异步下单
  - 排队系统
  - 答题降频
- [ ] **页面静态化与按钮置灰**

**面试题示例**：
- 设计一个百万 QPS 的秒杀系统
- 秒杀场景下，Redis 和 MySQL 数据如何保证一致？
- 秒杀完成后，库存数据怎么落库？

### 3.7 优惠券 / 营销系统（第 11 周）

- [ ] 优惠券模型设计（券模板 + 用户券）
- [ ] 防超发（领券限流）
- [ ] 优惠规则引擎（满减、折扣、券叠加）
- [ ] 价格计算引擎
- [ ] AB 测试与灰度

### 3.8 物流与售后（第 12 周）

- [ ] 物流轨迹查询（缓存策略）
- [ ] 第三方物流对接（顺丰 / 京东 / 菜鸟）
- [ ] 退换货流程
- [ ] 评价系统设计

## 五、实战项目推荐

### 5.1 开源电商项目（按推荐度排序）

| 项目 | Stars | 技术栈 | 适合人群 |
|-----|-------|--------|---------|
| **macrozheng/mall** | 75k+ | Spring Boot + MyBatis + Redis + ES + RabbitMQ | 入门首选 |
| **macrozheng/mall-swarm** | 12k+ | Spring Cloud Alibaba 微服务版 | 进阶 |
| **YunaiV/ruoyi-vue-pro** | 25k+ | Spring Boot 单体，业务完整 | 中小厂面试 |
| **shenzhuan/mallplus** | 6k+ | 多端商城 | 全栈了解 |
| **opentiny/tiny-engine** | - | 低代码 | 拓展视野 |

### 5.2 学习方式建议

1. **第一遍**：跑通项目 → 看懂目录结构 → 理解模块划分
2. **第二遍**：深入 1-2 个核心模块（订单、库存、秒杀）→ 画时序图 + 类图
3. **第三遍**：自己实现一个核心模块 → 加入自己的优化思考
4. **第四遍**：模拟"如果我是架构师，我会怎么设计"

### 5.3 自建项目思路

如果时间充裕，强烈建议**自己写一个 mini 电商**，规模可以小，但要五脏俱全：

```
mini-mall
├── user-service          用户与认证
├── product-service       商品与搜索
├── cart-service          购物车
├── order-service         订单
├── payment-service       支付（mock）
├── inventory-service     库存
├── seckill-service       秒杀
├── gateway               网关
└── common                公共组件
```

**技术选型建议**（Java 方向）：
- Spring Boot 3 + Spring Cloud Alibaba
- MySQL 8 + ShardingSphere
- Redis 7 + Redisson
- RocketMQ 5
- Elasticsearch 8
- Nacos + Sentinel
- Docker + Docker Compose

### 6.2 场景设计题模板（STAR 法）

回答系统设计题的"万能模板"：

```
1. 需求澄清（Scenario）
   - QPS 多少？数据量多少？读写比？
   - 一致性要求？延迟要求？

2. 容量估算（Capacity）
   - 存储 = 单条 × 数量
   - QPS = 日活 × 人均请求数 / 86400 × 峰值系数

3. 整体架构（Architecture）
   - 接入层 → 网关 → 业务层 → 缓存 → 存储

4. 核心流程（Flow）
   - 画时序图，标出关键节点

5. 数据模型（Data）
   - 表结构、索引、缓存 key 设计

6. 难点突破（Highlight）
   - 高并发：缓存 + 限流 + 异步
   - 一致性：分布式事务方案
   - 高可用：集群 + 熔断 + 降级

7. 监控与扩展（Operation）
   - 监控指标
   - 容量规划
   - 演进方向
```

## 十、最后的话

> **学习电商不是为了"背"，而是为了"懂"。**
>
> 每学一个知识点，问自己三个问题：
> 1. **为什么需要它？**（解决什么问题）
> 2. **不用它会怎样？**（替代方案对比）
> 3. **它的边界在哪？**（什么场景不适用）
>
> 把每个技术点都串到电商的业务场景里，面试时才能讲出"有血有肉"的答案，
> 而不是干巴巴的八股文。

**祝面试顺利，早日上岸！** 🚀
